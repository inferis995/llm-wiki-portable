# LLM Wiki Portable — Technical Knowledge Base

You are the maintainer of this wiki. You write, update, and organize all content. The user reads, sources, and directs.

## Wiki Root

All paths are relative to the wiki root directory. The user will specify the drive/path when starting a session (e.g. `D:\`, `E:\`, or a local folder).

## Prerequisites

- **RTFM MCP** — Optional but recommended for semantic search. Use `rtfm_search` (corpus `wiki`), `rtfm_expand`, and `rtfm_sync` after every save.

## Structure

```
{wiki-root}/
├── raw/              ← Source documents (IMMUTABLE — never modify)
│   └── assets/       ← Downloaded images
├── wiki/             ← LLM-generated markdown (you own this)
│   ├── index.md      ← Catalog of every page
│   ├── log.md        ← Append-only chronological record
│   └── *.md          ← Entity, concept, summary, comparison pages
├── web/              ← Static web UI (open web/index.html in browser)
│   └── data.json     ← Generated by sync.py
├── sync.py           ← Run after changes: python sync.py
└── CLAUDE.md         ← This file
```

## Operations

### Ingest
When the user provides a new source:
1. Save it to `raw/` (or read from URL/path)
2. Read the full source carefully
3. Discuss key takeaways with the user
4. Create/update wiki pages:
   - Summary page for the source
   - Update relevant entity/concept pages with new info
   - Add cross-references with `[[wikilinks]]`
   - Note contradictions with existing content
5. Update `wiki/index.md`
6. Append entry to `wiki/log.md`
7. Run `python {wiki-root}/sync.py` to regenerate web data
8. If RTFM MCP available: run `rtfm_sync` on the wiki directory

### Query
When the user asks a question:
1. Search the wiki (RTFM MCP if available, otherwise read index.md)
2. Read relevant pages
3. Synthesize answer with `[[page]]` citations
4. If the answer is valuable, offer to file it as a new wiki page

### Lint
When the user asks to health-check:
1. Scan all pages for contradictions, orphans, missing cross-references
2. Report findings and offer to fix

## Page Format

Every wiki page uses this format:

```markdown
---
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: [[source-page-1]], [[source-page-2]]
tags: [tag1, tag2]
---

# Page Title

Content with [[wikilinks]] to other pages.

## Key Points
- Point 1
- Point 2

## Related
- [[related-page-1]]
- [[related-page-2]]
```

## Conventions

- Use `[[wikilinks]]` for all cross-references
- Source pages go in `wiki/sources/` with prefix `src-`
- Entity pages go in `wiki/entities/`
- Concept pages go in `wiki/concepts/`
- Comparison pages go in `wiki/comparisons/`
- Be concise — prefer bullet points over paragraphs
- Always cite sources with `[[links]]`
- When in doubt, create a new page rather than cramming info into an existing one
- Content language: match the user's language

## Web UI

After any change, run `python {wiki-root}/sync.py` and open `web/index.html` in a browser to see the graph and pages.

## Log Format

`wiki/log.md` entries use this format:

```
## [YYYY-MM-DD] ingest | Source Title
- Created: [[src-source-name]], [[entity-1]], [[concept-1]]
- Updated: [[existing-page]] with new info
```
